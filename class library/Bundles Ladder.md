# Bundles Ladder

This course uses a **Bundles Ladder** grading system.

Instead of individual points for every assignment, grades reflect **consistent progress and reliability across the semester**.

Higher bundles require maintaining the standards of the lower bundles.

---

# Bundle 1 — Basic Participation

Teams at this level:

- submit Sprint Packets
- maintain a repository
- attempt demos

Projects may still be unstable or incomplete.

This bundle shows **participation but limited reliability**.

---

# Bundle 2 — Functional Progress

Teams at this level:

- submit consistent Sprint Packets
- demonstrate working features
- maintain visible GitHub workflow

Projects demonstrate **clear progress across multiple sprints**.

---

# Bundle 3 — Reliable System

Teams at this level:

- maintain reliable demos
- produce clean documentation
- show strong GitHub workflow

Projects show **engineering discipline and stability**.

---

# Bundle 4 — Professional Quality

Teams at this level:

- demonstrate a polished final system
- maintain clear documentation
- show consistent collaboration

Projects resemble **professional software work**.

---

# Evidence Matters

Bundles are determined using:

- Sprint Packets
- demo reliability
- GitHub activity
- documentation quality
- individual receipts

---

# Key Idea

The goal is not perfection.

The goal is **consistent improvement across the semester**.

Teams that show steady progress will reach higher bundles.
