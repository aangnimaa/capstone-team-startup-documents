# POLICY — AI Use

## Purpose

Artificial Intelligence tools are widely used in modern software development.

This course allows AI tools, but students must use them **responsibly and transparently**.

The goal is to support learning, not replace it.

---

# Allowed Uses

Students may use AI tools for:

- brainstorming ideas
- explaining concepts
- debugging assistance
- writing documentation
- generating starter code
- improving clarity of writing

AI can be a helpful assistant, but it should **not replace your understanding** of the project.

---

# Responsibility Rule

If you submit code or documentation, you must be able to:

- explain how it works
- explain why it exists
- explain how it connects to the rest of the project

If you cannot explain it, you should not submit it.

---

# Transparency

When AI is used in an important way, note it briefly.

Example in a Pull Request:

"Initial structure generated with AI and then modified by the team."

The goal is transparency, not punishment.

---

# Not Allowed

The following behaviors are not acceptable:

- submitting AI‑generated code you do not understand
- copying entire systems without review
- using AI to avoid doing assigned work
- submitting work produced entirely by AI without contribution

---

# AI Is a Tool, Not a Shortcut

Professional engineers use AI tools carefully.

They:

- test generated code
- review it carefully
- adapt it to their project

Students in this course should do the same.

---

# If You Are Unsure

If you are unsure whether AI use is acceptable, ask the instructor.

The goal is **learning and professional development**, not punishment.
